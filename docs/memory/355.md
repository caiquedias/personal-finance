# #355 — fix: enum enviado como string quebra deserialização no POST /expenses/batch/create
Issue: #355 | Data: 2026-06-26 | Branch: fix/355-enum-string-deserialization

## Arquivos criados
- `src/PersonalFinance.Api/Converters/FlexibleEnumConverterFactory.cs` — JsonConverterFactory global: deserializa enum a partir de int, string numérica ou nome (case-insensitive); serializa como int
- `tests/PersonalFinance.Api.Tests/Unit/Converters/FlexibleEnumConverterFactoryTests.cs` — 6 testes unitários cobrindo todos os caminhos de leitura e escrita

## Arquivos modificados
- `src/PersonalFinance.Api/Program.cs` — adicionado `new FlexibleEnumConverterFactory()` em `AddJsonOptions` para registro global
- `personal-finance/src/app/shared/components/modal/batch-expenses-modal/batch-expenses-modal.component.html` — 8 ocorrências `[value]` → `[ngValue]` em options de `SourceType` e `FortnightType` (newForm + editForm)

## Decisões técnicas
- `FlexibleEnumConverterFactory` registrada via `AddJsonOptions` (afeta apenas controllers, não o `ExceptionMiddleware`)
- Nested class `FlexibleEnumConverter<TEnum>` como `private sealed` dentro da factory — não viola "um arquivo por classe"
- `CanConvert` retorna `false` para `Nullable<TEnum>` — campos nullable só existem em `[FromQuery]`, nunca em JSON bodies
- Serialização mantida como integer para não quebrar comparações TypeScript no frontend
