## Decisões técnicas
- `FakePurgeRepository` usa `AppDbContext` InMemory ao invés de `List<>` local: `List<>` perde estado entre requests HTTP distintas no mesmo teste de integração
- `PurgeController.ExportPeriod` faz lookup de período no controller (antes de chamar use case) para extrair Year/Month e montar o nome do arquivo CSV no `Content-Disposition`
- `PurgeController.Purge` faz pre-check de ownership via `IPeriodRepository.GetByIdAndUserAsync` e lança `KeyNotFoundException` (→ 404) antes de chamar `PurgePeriodUseCase`, evitando que `DomainException` do use case retorne 400 para recurso não encontrado
- `DeleteRecord` usa double-query (GetAll + DeleteById) para validar ownership e retornar 404 — tech debt identificado pelo Reviewer, não bloqueia PR

## Desvios do spec
- Nenhum desvio funcional. `GetEligiblePeriods` lista períodos com `IsActive = false` via `IPeriodRepository` (sem use case dedicado — query simples no controller)
