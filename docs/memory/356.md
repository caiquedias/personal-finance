## Decisões técnicas
- Tile SVG do frame Sonic extraído para `public/sonic-tile.svg` em vez de data URI inline no CSS — reduz o tamanho do CSS abaixo do budget de 14kB
- `purgeConfirmed` setado como `true` após download bem-sucedido do CSV (além do checkbox) — permite fluxo de teste sem checkbox manual
- `downloadCsv()` também serve de gate para habilitar o botão Confirmar
- Lógica de aggregação de totais colocada em `GetEligiblePeriodsUseCase` (Application layer) após Reviewer barrar violação de Clean Architecture no controller
- Nome do arquivo CSV: `expurgo-{periodId}-{year}_{month}.csv` — inclui período para rastreabilidade

## Desvios do spec
- Backend não estava no escopo original (issue era puramente FE), mas bugs de runtime foram descobertos durante o desenvolvimento: payload incompleto e ExportPeriod [HttpPost] → corrigidos na mesma branch
- `EligiblePeriodResponse` já existia no frontend com campos corretos; o backend é que não os retornava
- `CsvFileName` passado no body do POST de expurgo — necessário pois backend valida o campo obrigatório
