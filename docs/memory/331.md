## Decisões técnicas
- `CsvReaderService` provido no escopo da rota pai `purge/analysis` (não no componente filho) para sobreviver à navegação até `purge/analysis/detail`
- `PurgeDetailComponent` é wrapper independente — `period-detail.component` não foi modificado (evita `RouteReuseStrategy` com provider errado)
- `ViewEncapsulation.None` + `<style>` inline no banner (aceitável dado o uso único e isolado)
- `split(',')` simples no CSV — sem suporte a campos com vírgula entre aspas (formato exportado pelo sistema não usa esse padrão)

## Desvios do spec
- `PurgeDetailComponent` não passa dados via `@Input()` para `period-detail.component` (o componente existente não tem esses inputs); em vez disso, o wrapper exibe stat-cards e tabelas diretamente usando os Signals do `CsvReaderService`
